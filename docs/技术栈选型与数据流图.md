## 二、技术栈选型分析

### 2.1 前端技术栈

#### 2.1.1 Vue 3

**选型理由：**
- 性能优异：采用Proxy响应式系统，性能比Vue 2提升1.3-2倍
- 组合式API：代码组织更灵活，逻辑复用更方便
- TypeScript支持：更好的类型推导和IDE支持
- 生态成熟：丰富的组件库和工具链
- 学习曲线平缓：适合快速开发

**对比其他框架：**
| 特性 | Vue 3 | React | Angular |
|------|-------|-------|---------|
| 学习难度 | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 开发效率 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 性能 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 生态系统 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 文档质量 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

#### 2.1.2 Vite

**选型理由：**
- 极速冷启动：基于ESM，开发服务器启动时间<1秒
- 热更新快：模块级别的HMR，更新速度极快
- 开箱即用：内置TypeScript、JSX、CSS预处理器支持
- 优化构建：基于Rollup，生产构建体积小
- 插件生态：丰富的插件系统

**对比Webpack：**
- 开发启动速度：Vite快10-100倍
- 热更新速度：Vite保持毫秒级，Webpack随项目增大变慢
- 配置复杂度：Vite更简洁

#### 2.1.3 Pinia

**选型理由：**
- 轻量级：体积仅1KB
- TypeScript友好：完整的类型推导
- 模块化：自动代码分割
- DevTools支持：完整的调试工具
- Vue 3官方推荐：替代Vuex


#### 2.1.4 ECharts

**选型理由：**
- 功能强大：支持关系图、力导向图等复杂可视化
- 性能优异：Canvas渲染，支持大数据量
- 交互丰富：缩放、拖拽、高亮等交互
- 文档完善：中文文档详细，示例丰富
- 社区活跃：百度开源，国内使用广泛

**应用场景：**
- 药物关系图谱可视化
- 相互作用统计图表
- 风险等级分布饼图

### 2.2 后端技术栈

#### 2.2.1 Koa 2

**选型理由：**
- 轻量级：核心代码仅几百行
- 中间件机制：洋葱模型，易于扩展
- async/await：原生支持，代码简洁
- 错误处理：统一的错误处理机制
- 灵活性高：不绑定任何中间件

**对比Express：**
| 特性 | Koa 2 | Express |
|------|-------|---------|
| 异步处理 | async/await | callback/promise |
| 中间件模型 | 洋葱模型 | 线性模型 |
| 错误处理 | try-catch | error middleware |
| 体积 | 更小 | 较大 |
| 灵活性 | 更高 | 内置功能多 |


#### 2.2.2 MongoDB + Mongoose

**选型理由：**
- 文档型数据库：适合存储非结构化的药物信息
- Schema灵活：易于扩展字段
- 查询强大：支持复杂查询和聚合
- 性能优异：读写速度快
- Mongoose ODM：简化数据库操作，提供数据验证

**数据模型优势：**
```javascript
// 药物信息可以包含数组、嵌套对象等复杂结构
{
  name: "阿司匹林",
  sideEffects: ["胃肠道不适", "出血风险"],  // 数组
  contraindications: ["消化道溃疡", "血友病"],
  metadata: {                                // 嵌套对象
    source: "ai",
    confidence: 0.95
  }
}
```

**对比关系型数据库：**
- 更适合存储药物的多值属性（副作用、禁忌症等）
- 无需复杂的JOIN操作
- Schema变更更灵活

#### 2.2.3 DeepSeek AI

**选型理由：**
- OpenAI兼容：使用标准API接口
- 性能优异：响应速度快，准确度高
- 成本合理：相比GPT-4更经济
- 中文友好：对中文药物名称理解准确
- 结构化输出：支持JSON格式返回

**应用场景：**
- 药物信息智能分析
- 相互作用风险评估
- 用药建议生成


#### 2.2.4 Swagger

**选型理由：**
- 自动生成文档：减少文档维护成本
- 交互式测试：可直接在文档中测试API
- 标准化：遵循OpenAPI规范
- 前后端协作：提供统一的接口定义
- 代码生成：可生成客户端SDK

## 四、关键技术实现

### 4.1 前端状态管理（Pinia）

```javascript
// stores/drugStore.js
import { defineStore } from 'pinia'

export const useDrugStore = defineStore('drug', {
  state: () => ({
    drugs: [],           // 药物列表
    currentDrug: null,   // 当前查看的药物
    loading: false,      // 加载状态
    error: null          // 错误信息
  }),
  
  actions: {
    async fetchDrugs(page = 1, limit = 10) {
      this.loading = true
      try {
        const response = await api.getDrugs(page, limit)
        this.drugs = response.data
      } catch (error) {
        this.error = error.message
      } finally {
        this.loading = false
      }
    }
  }
})
```

**数据流：**
```
组件调用action → 发起API请求 → 更新state → 视图自动更新
```

### 4.2 后端中间件机制（Koa）

```javascript
// 洋葱模型示意
app.use(async (ctx, next) => {
  console.log('1. 请求进入')
  await next()
  console.log('6. 响应返回')
})

app.use(async (ctx, next) => {
  console.log('2. 日志记录')
  await next()
  console.log('5. 日志记录')
})

app.use(async (ctx, next) => {
  console.log('3. 业务处理')
  ctx.body = { data: 'result' }
  console.log('4. 业务完成')
})

// 执行顺序：1 → 2 → 3 → 4 → 5 → 6
```

**中间件流程：**
```
请求 → CORS → 日志 → 错误处理 → 路由 → 控制器 → 响应
```

### 4.3 数据库查询优化（MongoDB）

```javascript
// 1. 索引优化
drugSchema.index({ name: 1 })           // 单字段索引
drugSchema.index({ name: 'text' })      // 全文索引
interactionSchema.index({ drug1Id: 1, drug2Id: 1 })  // 复合索引

// 2. 聚合查询
const stats = await Interaction.aggregate([
  { $match: { severity: 'high' } },     // 筛选
  { $group: {                           // 分组
      _id: '$severity',
      count: { $sum: 1 }
    }
  },
  { $sort: { count: -1 } }              // 排序
])

// 3. 投影优化（只查询需要的字段）
const drugs = await Drug.find()
  .select('name category')              // 只返回name和category
  .limit(10)
```

**查询性能优化：**
- 使用索引：查询速度提升10-100倍
- 投影字段：减少网络传输
- 分页查询：避免一次加载大量数据

### 4.4 AI接口集成（DeepSeek）

```javascript
// services/aiService.js
import OpenAI from 'openai'

const client = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: 'https://api.deepseek.com'
})

async function analyzeDrug(drugName) {
  const prompt = `你是一位专业的临床药师。请分析药物"${drugName}"...`
  
  const response = await client.chat.completions.create({
    model: 'deepseek-chat',
    messages: [{ role: 'user', content: prompt }],
    temperature: 0.7,
    response_format: { type: 'json_object' }  // 强制JSON输出
  })
  
  return JSON.parse(response.choices[0].message.content)
}
```

**AI调用流程：**
```
构建提示词 → 调用API → 解析响应 → 验证数据 → 保存数据库
```
