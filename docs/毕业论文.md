## 第1章 绪论

### 1.1 研究背景

随着信息技术的迅速发展及医疗信息化程度的提高，医疗行业焕发了新的生机。推动医疗健康发展的一个重要因素就是信息技术，药物信息系统就是信息技术在医疗行业应用的一个体现。国外已经有比较先进的药物信息管理系统，并且在各医院得到了普及。

### 2.2 Vue3框架介绍

Vue3是一种用于构建用户界面的渐进式JavaScript框架，是Vue.js的最新版本。Vue3引入了组合式API（Composition API），使得代码组织更加灵活，同时提高了TypeScript支持。

#### 2.2.1 组合式API

组合式API是Vue3的核心特性，主要包括以下内容：

（1）**ref和reactive**：ref用于创建响应式的基本类型数据，reactive用于创建响应式的对象类型数据。

```typescript
import { ref, reactive } from 'vue'

// ref用于基本类型
const count = ref(0)
count.value++

// reactive用于对象
const state = reactive({
  name: '张三',
  age: 25
})
```

（2）**computed计算属性**：用于基于响应式数据计算派生值。

```typescript
import { computed } from 'vue'

const firstName = ref('张')
const lastName = ref('三')
const fullName = computed(() => firstName.value + lastName.value)
```

（3）**watch和watchEffect**：用于监听响应式数据的变化。

```typescript
import { watch, watchEffect } from 'vue'

watch(count, (newValue, oldValue) => {
  console.log(`count从${oldValue}变为${newValue}`)
})

watchEffect(() => {
  console.log(`count的值为${count.value}`)
})
```

### 2.3 Node.js与Koa框架介绍

#### 2.3.1 Koa框架

Koa是由Express原班人马打造的 expressive middleware for node.js，它使用async函数实现优雅的中间件编写方式，相比Express更加轻量和现代化。

```javascript
import Koa from 'koa'
import Router from '@koa/router'

const app = new Koa()
const router = new Router()

// 中间件
app.use(async (ctx, next) => {
  console.log('请求开始')
  await next()
  console.log('请求结束')
})

// 路由
router.get('/api/drugs', async (ctx) => {
  ctx.body = { drugs: [] }
})

app.use(router.routes())
app.use(router.allowedMethods())

app.listen(3000)
```

### 3.1 系统需求分析

#### 3.1.1 功能需求分析

药物相互作用查询系统的设计主要包括以下模块：

（1）**药物查询模块**：用户可以输入药物名称查询药物的详细信息，包括药物名称、分类、描述、副作用、禁忌症等。

用户通过系统进行药物信息查询，同时可以快速查询药物的详细信息。为了保证每个查询结果的准确性，系统采用权威的药物数据库。用户还可以对药物相互作用进行检测，系统会根据药物数据给出风险评估和建议。

系统在用户登录的时候，对账号的合法性进行判断，确保系统的安全性。系统管理员可以管理药物数据信息，保证药物数据的准确性和时效性。

### 3.3 系统设计目标

本系统的设计目标如下：

（1）提供快速、准确的药物信息查询服务，减少用户查找药物手册的时间。

### 5.4 后端API实现

#### 5.4.1 药物查询API

```javascript
// src/routes/drugs.js
import Router from '@koa/router'
import Drug from '@/models/Drug'

const router = new Router()

// 获取药物列表
router.get('/api/drugs', async (ctx) => {
  const { search } = ctx.query
  const drugs = await Drug.find(search ? {
    $or: [
      { name: new RegExp(search, 'i') },
      { genericName: new RegExp(search, 'i') }
    ]
  } : {})
  ctx.body = { success: true, data: drugs }
})

// 获取单个药物详情
router.get('/api/drugs/:id', async (ctx) => {
  const drug = await Drug.findById(ctx.params.id)
  ctx.body = { success: true, data: drug }
})

// 创建药物
router.post('/api/drugs', async (ctx) => {
  const drug = await Drug.create(ctx.request.body)
  ctx.body = { success: true, data: drug }
})

// 更新药物
router.put('/api/drugs/:id', async (ctx) => {
  const drug = await Drug.findByIdAndUpdate(
    ctx.params.id,
    ctx.request.body,
    { new: true }
  )
  ctx.body = { success: true, data: drug }
})

// 删除药物
router.delete('/api/drugs/:id', async (ctx) => {
  await Drug.findByIdAndDelete(ctx.params.id)
  ctx.body = { success: true }
})

export default router
```

export const initGraph = (nodes, edges, container) => {

      }

    series: [{
      type: 'graph',
      layout: 'force',
      symbolSize: 50,
      roam: true,
      label: {
        show: true,
        position: 'right',
        formatter: '{b}'
      },
      edgeSymbol: ['circle', 'arrow'],
      edgeSymbolSize: [4, 10],
      data: nodes.map(node => ({
        name: node.name,
        id: node._id,
        category: node.category
      })),
      links: edges.map(edge => ({
        source: edge.source,
        target: edge.target,
        description: edge.description,
        lineStyle: {

      emphasis: {
        focus: 'adjacency',

      },

系统测试是软件测试的一个重要环节，其目的在于对整个软件系统进行全面测试，以确认软件系统是否满足客户需求和预期目标，同时也旨在发现软件中存在的问题，以便及时修复和改进。

系统测试的意义：

（1）可以确认软件系统是否满足客户需求和预期目标。系统测试是在完成各个模块的单元测试和集成测试之后进行的，通过对整个系统的测试，可以确认系统是否满足客户的要求和预期目标。

（2）发现软件中存在的问题。系统测试是在软件开发过程的最后阶段进行的，可以发现前期测试未发现的潜在缺陷和错误，有助于快速定位问题并及时修复。

（3）验证软件的功能和性能。系统测试可以确保软件的功能和性能完全符合规格说明和用户需求，并且在各种条件下都能正常工作。

（4）提高软件质量和可靠性。通过系统测试，可以尽早发现和修复存在的问题，同时还可以帮助开发团队不断改进和优化软件，从而提高软件的质量和可靠性，满足用户需求。

### 6.2 测试方法

#### 6.2.1 功能测试

对于一款软件而言，正确性尤为重要，因此功能测试是系统测试环节中必不可少的一部分。

（1）首先是用户登录界面，包括普通用户登录和管理员登录两部分，为了查询不同权限登录系统后各个功能是否正常，可以登录两部分的账号运行系统检测各个控件能否正常工作。

（2）接下来，测试药物查询功能，输入药物名称，看是否能正确返回药物信息。

（3）然后，测试相互作用检测功能，选择多种药物，看是否能正确检测出相互作用关系。

（4）最后，测试AI问答功能，输入药物相关问题，看是否能正确回答。

#### 6.2.2 界面测试

界面测试有七大黄金原则，分别是规范性、安全性、易用性、合理性、独特性、美观性、协调性。每个系统开发者希望最终呈现给用户的使用界面是简洁美观、规范整齐、便于操作的。因此，在进行界面测试时，要侧重于这些方面：合理利用空间，比例要协调，布局要合理，风格要一致。

### 7.1 论文总结

本文旨在介绍基于Vue+Node.js的药物相互作用查询系统的设计与实现的全过程，以浏览器层、Node.js服务器层、数据库服务器层三层体系架构为基础，选择Vue3为前端框架，Node.js+Koa为后端服务，MongoDB为后台数据库，结合药物相互作用查询需求，设计出一个药物相互作用查询系统。本人所做的主要工作如下：

（1）学习了Vue3组合式API开发，熟悉并练习了Node.js和Koa框架后端开发，了解了MongoDB数据库连接的具体步骤，安装了该系统设计所需要的平台和软件。

### 7.2 问题与展望

本系统能满足用户和医疗机构的初步需求，但是也存在功能上的不足，期待自己在未来可以对该系统进行完善。本系统存在的不足有：

（2）没有实现用户登录注册功能，后续可以添加该功能，使系统更加完善。

（3）用户界面相对简单，希望未来可以加以改善，继续优化界面，使其简洁且完整。

（4）AI问答功能依赖第三方API，未来可以考虑使用本地部署的大模型来提高响应速度和稳定性。
